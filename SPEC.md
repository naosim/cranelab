# CraneLab 開発仕様書

## 1. 概要
本プロジェクトは、クレーンゲーム（3本爪/確率機タイプ）の実戦的な技（「クルリンパ」「タグかけ」等）を練習できる、ブラウザ完結型の3Dシミュレータである。物理エンジンによるリアルな挙動の再現と、ユーザーによる詳細なパラメータ調整機能を最大の特徴とする。

## 2. 動作環境・技術スタック
* 配信プラットフォーム: GitHub Pages（静的ホスティング）
* 開発言語: TypeScript
* 3Dレンダリング: Three.js
* 物理エンジン: Rapier.js (`@dimforge/rapier3d-compat` / WASM版)
* ビルドツール: Vite
* 外部ファイル依存: なし（サウンドまでWeb Audio APIで生成）

## 3. システムアーキテクチャ（クラス構成）

### 3.1 GameManager
* 役割：ゲーム全体のライフサイクル管理、パラメータ変更の遅延適用、物理ステップとレンダリングのループ制御、景品リセット処理。

### 3.2 InputManager
* 役割：キーボード（矢印キー / QE / スペース / V）およびモバイルUIボタンからの入力を統合し、クレーン操作コマンドへ変換。

### 3.3 StageManager
* 役割：筐体（床、壁、シールド、枠）とドロップゾーン（深い落とし口・筒状壁）の生成。シールド高さの動的変更。

### 3.4 CraneController
* 役割：3本爪アームのXYZ移動、自動シーケンス（下降→閉じ→上昇→移動→開放）の制御、ジョイントモーター駆動による開閉とトルク制御、衝突制限、ファンファーレ判定。

### 3.5 PrizeFactory
* 役割：景品の生成。2種類のぬいぐるみ（立った犬／座った熊）をランダムに選択。タグ（六角形リング）の構築。ランダムY軸回転。

### 3.6 AudioManager
* 役割：Web Audio API を用いた効果音の生成。`clawClose`, `clawOpen`, `dropPrize`, `buttonClick`, `fanfare` の5種類。

### 3.7 ColliderDebug
* 役割：全コライダーのワイヤーフレーム表示。Vキーでトグル。

### 3.8 ControlPanel
* 役割：14個のスライダー＋1個のトグルスイッチによる全パラメータのリアルタイム調整。折りたたみ可能。Pキー／ギアボタンで開閉。localStorageへの自動保存。

### 3.9 MobileControls
* 役割：スマートフォン向けの十字キー（D-pad）と円形アクションボタンの提供。

## 4. 物理・オブジェクト設計

### 4.1 座標系
* 上方向: +Y
* 手前方向（カメラ側/プレイヤー側）: +Z
* シールド位置: `z = 0.85`
* プレイエリア: `x ∈ [-0.85, 0.85]`, `z ∈ [-0.85, 0.85]`, `y ∈ [0.08, 1.4]`

### 4.2 3本爪アームの構造
* クレーンヘッド（基部）: KinematicPositionBased の RigidBody。XYZ軸を移動。
* アームパーツ（3セット）:
  * クレーンヘッドを中心に、Y軸周りに120度ずつ等間隔に配置。
  * 1アーム＝1個の動的RigidBody。その上に3つのコライダー（上腕・前腕・爪）を直列に配置。
  * クレーンヘッドとの接続: RevoluteImpulseJoint（モーター駆動、ForceBased）。
  * ジョイント制限: `[maxCloseAngle, maxOpeningAngle + 0.1]`。
  * 前腕（forearm）・爪（claw）のコライダーは毎フレーム `setTranslationWrtParent` / `setRotationWrtParent` で視覚メッシュに追従。
  * 上腕コライダーは短め（視覚の 0.4倍）に設定し、誤衝突を防止。
  * コライダーサイズ: 上腕 `(0.02, ul*0.8, 0.02)`、前腕 `(0.02, fl*2, 0.02)`、爪 `(0.02, cl*2, 0.02)` の直方体。
* 開閉制御: 目標角度 `maxCloseAngle + currentOpenRatio * (maxOpeningAngle - maxCloseAngle)` をモーターに設定。
* 衝突グループ: `GROUP_ARM = 1`, `GROUP_PRIZE = 2`。アームと景品は衝突するがアーム同士は衝突しない。シールド・壁・床とは通常の衝突。

### 4.3 景品（ぬいぐるみ＋タグ）の構造
* 1個の動的RigidBodyに複数のコライダーを直接配置（Fixed Joint不使用）。
* **タイプ1: 立った犬（dog）**
  * 球体（胴体・頭・尾）+ カプセル（4本脚）の複合体。
  * 各コライダーは `setCollisionGroups(PRIZE_COLLISION_GROUPS)` でアームとの衝突を許可。
* **タイプ2: 座った熊（sitting_bear）**
  * 直方体（四角柱）のみで構成。胴体・頭・腕（左右）・脚（左右）。
  * 腕と脚は体に対して前方（+Z）にオフセット。
* 共通:
  * 初期位置: `(0, 0.7, 0)`。PIVOT_Y = 0.7。
  * 生成時にランダムなY軸回転。
  * CCD（連続的衝突検出）有効。
  * 線形減衰 2.0、角度減衰 1.5。

### 4.4 タグ（六角形リング）
* 6個の直方体コライダーによる六角形の輪。
* ぬいぐるみボディ上に直接構築（別ボディ＋Fixed Jointではない）。
* 水平（XZ平面）に配置。`rotation.x = π/2`。
* オフセット位置: `(0, 0.03*S, -0.18*S)`（S=2で `(0, 0.06, -0.36)`）。
* 視覚表現: TorusGeometry に同様の回転・位置。
* 摩擦: 1.0（タグかけ時の保持力を再現）。
* 衝突グループは景品本体と同じ。

### 4.5 ドロップゾーン（落とし口）
* シールド直後（`z ≈ 0.95〜1.75`）に位置する深いピット。
* 床: `y = -1.75`（5倍深度）。
* 4面の壁で完全な筒状（前面・左面・右面・背面）。
* 壁の上端は `y = 0`（地表）。正のY領域には壁が出ない。
* アームは MOVING_OUT 時に `y ≈ 1.4` を移動するため壁に衝突しない。

### 4.6 シールド
* 可変高さの直方体。`z = 0.85`（プレイエリア前端）に位置。
* 色 `#ff8844`、透明度 0.55。
* 高さ変更時にボディ・メッシュ・ジオメトリを動的更新。
* 衝突制限（下降速度制限）の対象に含まれる。

### 4.7 壁（筐体）
* 背面壁（`z = -0.85`）: 色 `#5599bb`、透明度 0.35。エッジライン付き。
* 側面壁（`x = ±0.85`）: 透明度 0.08。
* すべて `depthWrite: false` で透過レンダリング。

## 5. 自動シーケンス

### 5.1 状態遷移
```
IDLE → DESCENDING → CLOSING → ASCENDING → MOVING_OUT → OPENING → IDLE
```

### 5.2 各フェーズの詳細
1. **DESCENDING（下降）**: アームを `MOVE_SPEED` で下降。`isArmContact` が接触を検出すると速度を `collisionLimitForce` 倍に減速。0.02m 押し込まれるか `CLAMP_MIN.y` に達したら CLOSING へ。
2. **CLOSING（閉爪）**: 0.8秒間かけて爪を閉じる。`currentOpenRatio` を `dt*4` で減少。
3. **ASCENDING（上昇）**: `MOVE_SPEED` で `CLAMP_MAX.y` まで上昇。
4. **MOVING_OUT（移動）**: `MOVE_SPEED_FAST`（0.6）で `DROP_POS (0, 1.2)` へ移動。
5. **OPENING（開放）**: 0.6秒間かけて爪を開く。この開始時の景品Y座標を記録。
6. **IDLE**: 待機状態に戻る。

### 5.3 ファンファーレ判定
* OPENING 開始時の景品Y座標 `openingPrizeY` を記録。
* 毎フレーム、現在の景品Y座標との差が `DROP_DELTA_Y`（0.15）を超えたらファンファーレ再生。
* 差が小さければ（景品が落ちなかった／掴めなかった）ファンファーレ無し。
* フラグ `fanfarePlayed` で重複再生を防止。次のアクション開始時にリセット。

### 5.4 衝突制限
* `collisionLimitEnabled` が ON のとき、`isArmContact` が「景品・ヘッド以外との接触」（壁・床・シールド・ピット壁等）を検出すると下降速度を制限。
* `exempt = new Set(["prize_bear", "crane_head"])` — 景品とヘッドは制限対象外。
* 手動操作中の上昇（moveY > 0）時も同様の制限がかかる。

## 6. パラメータ設計（ユーザー調整機能）

画面上のGUI（ControlPanel）から以下のパラメータをリアルタイムに変更可能。localStorageに自動保存。

### 6.1 筐体・ステージ設定
| パラメータ | キー | 範囲 | 説明 |
|-----------|------|------|------|
| シールド高さ | shieldHeight | 0〜1.0 | 落とし口の壁高。ぬいぐるみの乗り越え難易度に直結。 |

### 6.2 アーム挙動・パワー設定
| パラメータ | キー | 範囲 | 説明 |
|-----------|------|------|------|
| つかみ強度 | closingTorque | 10〜1000 | 爪を閉じる際の最大トルク。 |
| ホールド力 | holdTorque | 1〜500 | 上昇中・移動中に維持するトルク（確率機のパワー抜け再現用）。 |
| 最大開き角 | maxOpeningAngle | 0〜1.5 | アーム最大展開角度（rad）。 |
| 閉じ切り角度 | maxCloseAngle | 0〜0.5 | 爪を完全に閉じたときの角度（rad）。 |

### 6.3 アーム形状
| パラメータ | キー | 範囲 | 説明 |
|-----------|------|------|------|
| 手首の角度 | clawPitch | 0〜0.8 | 爪先の内向き角度。タグへの引っかかりに影響。 |
| ひじの角度 | forearmAngle | 0〜2.0 | 前腕の折れ曲がり角度。 |
| 上腕長さ | upperArmLength | 0.1〜0.5 | 上腕の長さ。 |
| 前腕長さ | forearmLength | 0.1〜0.5 | 前腕の長さ。 |
| 手（爪）長さ | clawLength | 0.03〜0.15 | 爪の長さ。 |
| アーム回転角 | armRotation | 0〜90（度） | アーム全体のY軸回転。度単位で指定し内部でradに変換。 |

### 6.4 景品・衝突設定
| パラメータ | キー | 範囲 | 説明 |
|-----------|------|------|------|
| 景品 質量 | prizeMass | 0.01〜5.0 | ぬいぐるみの質量。重量が変わると掴みにくさに影響。 |
| 衝突制限 | collisionLimitEnabled | ON/OFF | アームが壁などに当たったときに下降を制限する。 |
| 衝突制限 強さ | collisionLimitForce | 0〜1.0 | 制限時の速度倍率（小さいほど強く制限）。 |
| アーム摩擦 | clawFriction | 0〜2 | 爪コライダーの摩擦係数。タグかけ時の滑りにくさ。 |

## 7. キーボード操作
| キー | 操作 |
|------|------|
| ↑↓←→ | アーム前後左右（XZ平面）移動 |
| Q / E | アーム下降・上昇（Y軸） |
| スペース | 爪を閉じる（手動時）/ アクション（自動シーケンス起動・飛ばし） |
| V | 衝突デバッグ表示トグル |
| P | コントロールパネル開閉 |

## 8. サウンド
すべて Web Audio API（OscillatorNode + GainNode）で生成。外部音声ファイル不要。
| サウンド | トリガー | 種類 |
|---------|----------|------|
| clawClose | CLOSING 開始時 | 200Hz square |
| clawOpen | IDLE 移行時 | 150Hz square |
| dropPrize | OPENING 開始時 | 80Hz sine |
| buttonClick | アクションボタン押下 | 800Hz square |
| fanfare | 景品がドロップゾーンに落下検出時 | 4音 sine（523/659/784/1047 Hz） |

## 9. 景品リセット
* 画面右上の「景品リセット」ボタンで現在の景品を削除し、新しい景品をランダムに生成。
* 景品の種類（dog / sitting_bear）とY軸回転は毎回ランダムに決定。

## 10. デバッグ機能
* ColliderDebug: Vキーで全Rapierコライダーのワイヤーフレームを表示。`ColliderDebug.update()` で毎フレーム更新。
* ファンファーレデバッグログ: OPENING 開始から3秒間、景品Y座標と落下量をコンソールに出力。
